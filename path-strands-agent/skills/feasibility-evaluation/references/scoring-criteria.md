# Feasibility 평가 항목별 세부 점수 기준

## 1. 데이터 접근성 (Data Accessibility) - 10점 만점

Agent가 필요한 데이터에 얼마나 쉽게 접근할 수 있는가?

| 점수 | 조건 | 예시 |
|------|------|------|
| **10점** | MCP 서버 또는 RAG 시스템 존재 | Notion MCP, 벡터 DB RAG |
| **9점** | REST/GraphQL API 존재 | OpenAPI 문서화된 API |
| **7점** | 파일 기반 접근 | CSV, JSON, Excel 파일 |
| **6점** | DB 직접 접근 | SQL 쿼리로 조회 가능 |
| **3점** | 화면 스크래핑 필요 | 웹 크롤링, 브라우저 자동화 |
| **0점** | 오프라인/수동 데이터만 | 종이 문서, 구두 정보 |

### 감점 요소
- API 인증 복잡도: -1점 (OAuth2 등)
- 데이터 실시간성 제약: -1점 (일배치 등)
- 데이터 품질 이슈: -1점 (결측, 비정규화)

---

## 2. 판단 명확성 (Decision Clarity) - 10점 만점

Agent가 의사결정을 내릴 때 얼마나 명확한 기준이 있는가?

| 점수 | 조건 | 예시 |
|------|------|------|
| **10점** | 명확한 if-then 규칙 | "금액 > 100만원 → 승인 필요" |
| **8점** | 레이블링된 예시 100+ | 분류된 학습 데이터 존재 |
| **6점** | 암묵적 패턴 존재 | 규칙은 있으나 문서화 안됨 |
| **4점** | 전문가 직감 의존 | "경험상 이런 경우는..." |
| **2점** | "그냥 알 수 있어" | 설명 불가능한 판단 |
| **0점** | 기준 없음 | 매번 다른 기준 적용 |

### 판정 가이드
- 명확한 분류 체계가 문서화되어 있는가?
- 예외 케이스 처리 규칙이 정의되어 있는가?
- 경계 케이스(애매한 경우) 처리 기준이 있는가?

---

## 3. 오류 허용도 (Error Tolerance) - 10점 만점

Agent가 실수했을 때의 영향과 수용 가능 수준

| 점수 | 조건 | 예시 |
|------|------|------|
| **10점** | 틀려도 괜찮음 | 내부 문서 초안 작성 |
| **8점** | 리뷰 후 실행 | 이메일 초안 → 검토 후 발송 |
| **5점** | 90%+ 정확도 필요 | 고객 문의 자동 분류 |
| **3점** | 99%+ 정확도 필요 | 계약서 조항 분석 |
| **0점** | 100% 정확도 필수 | 의료 진단, 법적 판단 |

### 리스크 레벨별 권장사항
- **낮음 (8-10점)**: 자동화 적합
- **중간 (5-7점)**: Human-in-the-Loop 권장
- **높음 (0-4점)**: AI 보조 역할로 제한

---

## 4. 지연 요구사항 (Latency Requirements) - 10점 만점

응답 시간 제약

| 점수 | 조건 | 예시 |
|------|------|------|
| **10점** | 몇 시간 OK | 야간 배치 리포트 |
| **9점** | 몇 분 OK | 이메일 자동 답변 |
| **7점** | 1분 이내 | 챗봇 응답 |
| **5점** | 10초 이내 | 실시간 추천 |
| **3점** | 3초 이내 실시간 | 검색 자동완성 |
| **0점** | 밀리초 단위 | 트레이딩 시스템 |

### LLM 응답 시간 참고
- Claude Opus: 10-30초 (복잡한 작업)
- Claude Sonnet: 3-10초 (일반 작업)
- 캐싱 활용 시: 1초 미만 가능

---

## 5. 통합 복잡도 (Integration Complexity) - 10점 만점

외부 시스템 연동 난이도

| 점수 | 조건 | 예시 |
|------|------|------|
| **10점** | 독립 실행 | 파일 입출력만 |
| **8점** | 1-2개 시스템 | Slack + Google Sheets |
| **5점** | 3-5개 시스템 | CRM + ERP + 이메일 + 캘린더 |
| **3점** | 레거시 시스템 | SOAP API, 메인프레임 |
| **1점** | 커스텀 프로토콜 | 독자 통신 규격 |
| **0점** | 미공개 시스템 | API 없는 폐쇄 시스템 |

### 감점 요소
- 방화벽/VPN 필요: -1점
- 인증 토큰 관리 복잡: -1점
- 트랜잭션 처리 필요: -1점
- 데이터 동기화 필요: -1점

---

## 6. 자율성 요구도 (Autonomy Requirement) - 10점 만점 (별도 축)

이 업무가 에이전트의 자율적 판단을 얼마나 필요로 하는가?

**주의: 이 항목은 위 5개 항목과 별도의 독립 축으로, 총 50점에 포함되지 않습니다.**

| 점수 | 조건 | 예시 |
|------|------|------|
| **9-10점** | 완전 자율: 예측 불가 상황에서 독립적 판단 | 자유형 리서치, 복잡한 문제 해결 |
| **7-8점** | 높은 자율: 다양한 경로 중 최적 선택 | 다변량 분석 후 의사결정, 동적 워크플로우 |
| **5-6점** | 중간: 일부 판단 필요, 대부분 정해진 흐름 | 예외 처리 포함 파이프라인 |
| **3-4점** | 낮은 자율: 거의 결정적, AI는 특정 단계만 보조 | 템플릿 기반 생성, 규칙 적용 |
| **0-2점** | 자율 불필요: 완전 결정적 파이프라인 | 포맷 변환, 단순 요약 |

### 판단 고려사항
- PROCESS 단계 간 동적 전환 필요 여부
- Human-in-Loop 수준 (완전 자동 = 높은 자율 경향)
- 판단 명확성과의 역관계 (명확한 규칙 = 낮은 자율)
- 예외 처리 복잡도
- 출력의 자유도 (자유형 > 구조화)

### 자동화 수준 매핑
| 자율성 점수 | 권장 자동화 수준 | 구현 방식 |
|-------------|-----------------|-----------|
| ≤5 | AI-Assisted Workflow | 워크플로우 엔진 + AI 단계 |
| ≥6 | Agentic AI | LLM 기반 자율 에이전트 |

**중요: 낮은 점수는 "나쁜" 것이 아닙니다.** 단순하고 예측 가능한 업무는 AI-Assisted Workflow로 더 안정적이고 효율적으로 구현할 수 있습니다.

---

## 종합 점수 계산

```
총점 = 데이터접근성 + 판단명확성 + 오류허용도 + 지연요구사항 + 통합복잡도
```

### 가중치 적용 (선택적)
특정 프로젝트에서 중요도가 다른 경우:

| 상황 | 가중치 강조 항목 |
|------|-----------------|
| 데이터 중심 | 데이터 접근성 x1.5 |
| 의사결정 중심 | 판단 명확성 x1.5 |
| 미션 크리티컬 | 오류 허용도 x1.5 |
| 실시간 서비스 | 지연 요구사항 x1.5 |
| 복잡한 환경 | 통합 복잡도 x1.5 |
