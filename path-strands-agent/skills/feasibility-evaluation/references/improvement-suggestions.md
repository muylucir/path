# 항목별 개선 제안 템플릿

취약 항목(7점 이하)에 대한 구체적인 개선 방안을 제시합니다.

---

## 1. 데이터 접근성 개선

### 현재 3점 이하 (스크래핑/오프라인)

**문제**: 데이터가 구조화되지 않았거나 프로그래밍 방식으로 접근 불가

**개선 방안**:
| 현재 상태 | 개선 방법 | 예상 점수 |
|-----------|-----------|-----------|
| 화면 스크래핑 | Playwright/Puppeteer 자동화 구축 | 3→5점 |
| 수동 엑셀 | Google Sheets API 연동 | 0→7점 |
| 이메일 기반 | 이메일 파싱 시스템 구축 | 0→6점 |
| 종이 문서 | OCR + 구조화 파이프라인 | 0→5점 |

**구체적 액션**:
1. 데이터 접근 빈도 분석 (일 1회? 실시간?)
2. API 존재 여부 재확인 (숨겨진 API 있을 수 있음)
3. 데이터 제공자에게 API 요청 검토
4. 중간 데이터 레이어 구축 검토 (ETL 파이프라인)

### 현재 6-7점 (DB/파일)

**문제**: 접근은 가능하나 효율적이지 않음

**개선 방안**:
| 현재 상태 | 개선 방법 | 예상 점수 |
|-----------|-----------|-----------|
| DB 직접 쿼리 | 읽기 전용 API 레이어 구축 | 6→8점 |
| 파일 기반 | 캐싱 + 인덱싱 시스템 | 7→9점 |
| 비정규 데이터 | 데이터 정규화 작업 | 5→7점 |

## 2. 판단 명확성 개선

### 현재 4점 이하 (전문가 직감/설명 불가)

**문제**: 의사결정 기준이 명시적이지 않음

**개선 방안**:
| 현재 상태 | 개선 방법 | 예상 점수 |
|-----------|-----------|-----------|
| 전문가 직감 | 의사결정 과정 인터뷰 + 문서화 | 4→6점 |
| 기준 없음 | 100건 예시 수집 + 레이블링 | 2→8점 |
| 암묵적 규칙 | 규칙 추출 워크숍 진행 | 6→8점 |

**구체적 액션**:
1. **인터뷰 진행**: 전문가와 30분 인터뷰, "왜 이렇게 판단했나요?" 질문
2. **예시 수집**: 최근 100건의 의사결정 사례 수집
3. **규칙 추출**: "이런 경우에는 X, 저런 경우에는 Y" 패턴 문서화
4. **경계 케이스 정의**: 애매한 경우 어떻게 처리할지 명시

### 개선 예시

**Before (4점)**:
> "경험 많은 담당자가 보면 알아요"

**After (8점)**:
> - 금액 100만원 초과 → 팀장 승인
> - 신규 거래처 → 신용 조회 필수
> - 긴급 요청 → 24시간 내 처리
> - 예외: [별첨 예외 목록 참조]

## 3. 오류 허용도 개선

### 현재 5점 이하 (높은 정확도 요구)

**문제**: 실수 시 비용/리스크가 큼

**개선 방안**:
| 현재 상태 | 개선 방법 | 예상 점수 |
|-----------|-----------|-----------|
| 100% 필요 | AI→사람 제안 방식으로 전환 | 0→5점 |
| 99% 필요 | 이중 검증 시스템 도입 | 3→6점 |
| 90% 필요 | 샘플링 검토 프로세스 | 5→7점 |

**Human-in-the-Loop 도입 시나리오**:
1. **검토 필수**: AI가 초안 생성, 사람이 최종 승인
2. **예외 에스컬레이션**: 신뢰도 낮은 경우만 사람에게 전달
3. **샘플 검토**: 무작위 10%만 사람이 검토

**리스크 완화 전략**:
- 실행 전 미리보기/시뮬레이션
- 되돌리기(Undo) 기능 구현
- 점진적 롤아웃 (1% → 10% → 100%)

## 4. 지연 요구사항 개선

### 현재 5점 이하 (실시간 요구)

**문제**: LLM 응답 시간이 요구사항에 미달

**개선 방안**:
| 현재 상태 | 개선 방법 | 예상 점수 |
|-----------|-----------|-----------|
| 3초 필요 | 결과 캐싱 + 프리페치 | 3→7점 |
| 10초 필요 | 병렬 처리 + 경량 모델 | 5→8점 |
| 1분 필요 | 비동기 처리 + 알림 | 7→9점 |

**최적화 기법**:
1. **캐싱**: 유사 질문 결과 재사용
2. **프리페치**: 예상 요청 미리 처리
3. **모델 선택**: 복잡도에 따라 Sonnet/Haiku 사용
4. **스트리밍**: 결과를 점진적으로 표시

**기대치 조정**:
- 사용자에게 "처리 중" 피드백 제공
- 부분 결과 먼저 표시 후 상세 결과 추가
- 배치 처리로 전환 검토

## 5. 통합 복잡도 개선

### 현재 5점 이하 (다수 시스템/레거시)

**문제**: 연동 대상이 많거나 레거시 시스템

**개선 방안**:
| 현재 상태 | 개선 방법 | 예상 점수 |
|-----------|-----------|-----------|
| 5+ 시스템 | 통합 API Gateway 구축 | 5→7점 |
| 레거시 API | 어댑터 레이어 구축 | 3→6점 |
| 커스텀 프로토콜 | 표준 API 래퍼 개발 | 1→5점 |

**아키텍처 개선**:
1. **API Gateway**: 다수 시스템을 단일 인터페이스로
2. **어댑터 패턴**: 레거시 시스템 추상화
3. **이벤트 버스**: 시스템 간 느슨한 결합
4. **MCP 서버**: 표준화된 도구 인터페이스 제공

**단계적 접근**:
1. 핵심 시스템 1-2개만 먼저 연동
2. 나머지는 수동/반자동으로 진행
3. 점진적으로 자동화 범위 확대
