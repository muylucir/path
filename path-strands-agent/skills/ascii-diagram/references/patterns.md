## 박스

### 기본 박스 (열린 스타일)
```
┌────────────────
│   제목
└────────────────
```

### 헤더가 있는 박스
```
┌────────────────────
│    시스템명
├────────────────────
│  컴포넌트 A
│  컴포넌트 B
│  컴포넌트 C
└────────────────────
```

### 가로선 스타일 (권장)
```
═══════════════════════════
  PRIMARY PATTERNS
═══════════════════════════
  ReAct        Tool Use
  (추론)       (도구)
───────────────────────────
  • Slow Query 분석
  • 담당자 추론
═══════════════════════════
```

### 연결된 박스 (가로)
```
[ 입력 ] ────▶ [ 처리 ] ────▶ [ 출력 ]
```

### 연결된 박스 (세로)
```
┌─────────────
│  Frontend
└─────┬───────
      │
      ▼
┌─────────────
│  Backend
└─────┬───────
      │
      ▼
┌─────────────
│  Database
└─────────────
```

### 아키텍처 박스
```
══════════════════════════════════════
  시스템 개요
══════════════════════════════════════

  [ Web Server ]  [ API Gateway ]  [ Worker ]
         │              │              │
         └──────────────┼──────────────┘
                        │
                        ▼
                  [ Database ]

══════════════════════════════════════
```

## 플로우차트

### 기본 분기
```
       [ 시작 ]
           │
           ▼
       < 조건? >
        │     │
       Yes    No
        │     │
        ▼     ▼
    [처리A] [처리B]
        │     │
        └──┬──┘
           ▼
       [ 종료 ]
```

### 다중 분기
```
          [ 입력값 ]
              │
    ┌─────────┼─────────┐
    │         │         │
    ▼         ▼         ▼
[Case 1] [Case 2] [Case 3]
    │         │         │
    └─────────┼─────────┘
              ▼
          [ 결과 ]
```

### 루프
```
    [ 시작 ]
        │
        ▼
   ┌──────────┐
   │          │
   ▼          │
[ 데이터 처리 ]│
   │          │
   ▼          │
< 완료? >─No──┘
   │
  Yes
   │
   ▼
[ 종료 ]
```

### 병렬 처리
```
        [ Coordinator ]
              │
    ┌─────────┼─────────┐
    │         │         │
    ▼         ▼         ▼
[Agent A] [Agent B] [Agent C]
    │         │         │
    └─────────┼─────────┘
              ▼
        [ Aggregator ]
```

## 시퀀스

### 기본 시퀀스
```
  User        System
    │           │
    │──요청────▶│
    │           │
    │◀──응답────│
    │           │
```

### 3자 통신
```
  Client     Server       DB
    │          │          │
    │──HTTP───▶│          │
    │          │──SQL────▶│
    │          │◀─결과────│
    │◀─JSON────│          │
    │          │          │
```

### 비동기 메시지
```
  Producer    Queue     Consumer
     │          │          │
     │──발행───▶│          │
     │◀ ─ACK─ ─ │          │
     │          │──수신───▶│
     │          │◀ ─완료─ ─│
     │          │          │
```

### Agent 통신
```
  User    Coordinator   Agent     Tool
    │          │          │         │
    │──요청───▶│          │         │
    │          │──위임───▶│         │
    │          │          │──호출──▶│
    │          │          │◀─결과───│
    │          │◀─완료────│         │
    │◀─응답────│          │         │
    │          │          │         │
```

## 트리

### 디렉토리 구조
```
project/
├── src/
│   ├── components/
│   │   ├── Header.tsx
│   │   └── Footer.tsx
│   └── pages/
│       └── index.tsx
├── public/
│   └── images/
└── package.json
```

### 조직도 (가로선 스타일)
```
                 [ CEO ]
                    │
     ┌──────────────┼──────────────┐
     │              │              │
 [ 개발팀 ]    [ 기획팀 ]    [ 운영팀 ]
     │              │              │
  백엔드          PM           인프라
  프론트         디자인          보안
```

### 카테고리 트리
```
서비스
├── 인증
│   ├── 로그인
│   ├── 회원가입
│   └── 비밀번호 찾기
├── 주문
│   ├── 장바구니
│   └── 결제
└── 마이페이지
    └── 프로필
```

### 의사결정 트리
```
문제 발생
├── [에러 로그 확인]
│   ├── DB 에러 → DB 재시작
│   ├── API 에러 → 재시도 로직
│   └── 시스템 에러 → 스케일 업
└── [로그 없음]
    └── 모니터링 추가
```

## 복합 예시

```
═══════════════════════════════════════
  PRIMARY PATTERNS
═══════════════════════════════════════

  ReAct          Tool Use       State Machine
  (추론)         (도구)         (상태 관리)

  • 원인 분석    • API 쿼리     • 에스컬레이션
  • 담당자 추론  • Slack 연동   • 타이머 전이

═══════════════════════════════════════
              │
              ▼
═══════════════════════════════════════
  COMPLEMENTARY PATTERNS
═══════════════════════════════════════

  Human-in-the-Loop (2가지 트리거)

  1. 매핑 실패 시 → 담당자 직접 지정
  2. 최종 단계   → 직접 처리

═══════════════════════════════════════
```
