# 파이프라인 설계 패턴

AI-Assisted Workflow에서 사용되는 주요 파이프라인 패턴입니다.

## 1. Sequential Pipeline (순차 파이프라인)

가장 기본적인 패턴. 각 단계가 순서대로 실행됩니다.

```
[입력] → [전처리] → [AI 분류] → [처리] → [AI 요약] → [출력]
```

**적합 상황:**
- 단계 간 의존성이 명확한 경우
- 각 단계의 입출력이 1:1 매핑
- 데이터 처리, 문서 변환, 보고서 생성

**구현 방식:**
- 워크플로우 엔진: 순차 State 연결
- 서버리스 함수 + 메시지 큐 연결
- 코드: async/await 체이닝

**예시 — 고객 문의 자동 응답:**
```
이메일 수신 → 본문 추출 → [AI: 문의 분류] → 템플릿 선택 → [AI: 응답 생성] → 검토 큐 → 발송
```

## 2. Fan-out / Fan-in (분산-수집)

하나의 입력을 여러 병렬 처리 후 결과를 합산합니다.

```
            ┌→ [AI 감성분석] ─┐
[입력] → [분할] → [AI 키워드추출] → [결합] → [출력]
            └→ [AI 요약]     ─┘
```

**적합 상황:**
- 동일 데이터에 여러 AI 분석을 병렬 수행
- 처리 시간 단축이 중요한 경우
- 독립적인 분석 결과를 종합해야 하는 경우

**구현 방식:**
- 워크플로우 엔진: Parallel State
- 서버리스 함수 + 메시지 큐: Fan-out 패턴
- 이벤트 버스: 이벤트 기반 병렬 트리거

**예시 — 문서 종합 분석:**
```
문서 업로드 → [분할] → AI 요약 / AI 핵심어 / AI 감성분석 → [종합] → 분석 보고서
```

## 3. Conditional Pipeline (조건부 파이프라인)

AI 판단 결과에 따라 다른 경로를 실행합니다.

```
                    ┌→ [긴급 처리] → [즉시 알림]
[입력] → [AI 분류] ─┤→ [일반 처리] → [배치 큐]
                    └→ [스팸 필터] → [폐기]
```

**적합 상황:**
- AI 분류 결과에 따라 후속 처리가 달라지는 경우
- 우선순위 기반 라우팅
- 예외 처리 경로가 필요한 경우

**구현 방식:**
- 워크플로우 엔진: Choice State + AI 분류 결과
- 이벤트 버스: 룰 기반 라우팅

**예시 — 티켓 자동 라우팅:**
```
티켓 접수 → [AI: 긴급도 판단] → 긴급(즉시 담당자 배정) / 보통(자동 응답 후 큐) / 스팸(폐기)
```

## 4. Event-driven Pipeline (이벤트 기반)

외부 이벤트에 의해 파이프라인이 트리거됩니다.

```
[이벤트 소스] → [이벤트 버스] → [전처리 함수] → [AI 처리] → [후처리] → [알림]
```

**적합 상황:**
- 실시간 반응이 필요한 경우
- 다양한 이벤트 소스 통합
- 느슨한 결합이 필요한 마이크로서비스 환경

**구현 방식:**
- 이벤트 버스 + 서버리스 함수 + LLM 서비스
- 메시지 큐(SNS/SQS) 기반 비동기 처리

**예시 — 실시간 로그 분석:**
```
로그 스트림 → 이벤트 룰 → [AI: 이상 탐지] → 정상(무시) / 이상(알림 + 티켓 생성)
```

## AI 통합 포인트 설계

파이프라인 내에서 AI를 활용하는 주요 단계:

### 분류/라우팅 단계
- 입력 데이터를 카테고리로 분류하여 다음 경로 결정
- 예: 고객 문의 → {환불, 배송, 기술지원} 분류

### 추출/변환 단계
- 비정형 데이터에서 구조화된 정보 추출
- 예: 이메일 본문 → JSON 구조 추출

### 생성/요약 단계
- 수집된 데이터를 바탕으로 텍스트 생성
- 예: 분석 결과 → 요약 보고서 생성

### 검증/평가 단계
- 이전 단계 결과의 품질 검증
- 예: 생성된 응답의 정확성/적절성 평가

## 각 AI 통합 포인트의 설계 항목

```
[Stage N] ──입력 스키마──▶ [AI 호출] ──출력 스키마──▶ [Stage N+1]
```

- **입력**: 이전 단계의 출력 + 컨텍스트 (JSON/텍스트)
- **프롬프트**: 태스크 지시 + 출력 포맷 지정 (단일 태스크, 구조화된 출력)
- **출력**: 구조화된 JSON (Structured Output 권장)
- **폴백**: AI 실패 시 재시도 → 기본값 → 사람에게 에스컬레이션

## 모델 선택 전략

| 태스크 유형 | 추천 모델 티어 | 이유 |
|-----------|:----------:|------|
| 간단한 분류/추출 | 경량 모델 | 빠르고 저렴, 구조화된 태스크에 충분 |
| 복잡한 분석/판단 | 중간 모델 | 성능과 비용의 균형 |
| 고품질 생성/요약 | 중간 모델 | 창의적 생성에 적합 |
| 극도로 복잡한 추론 | 대형 모델 | 최고 품질, 비용 높음 |

**비용 최적화**: 분류/추출 단계는 경량 모델, 생성/요약 단계는 중간 이상 모델을 사용하여 단계별로 모델을 다르게 선택합니다.

## 패턴 선택 가이드

| 요구사항 | 추천 패턴 |
|---------|----------|
| 단순 순차 처리 | Sequential |
| 병렬 분석 필요 | Fan-out/Fan-in |
| AI 판단에 따른 분기 | Conditional |
| 외부 이벤트 반응 | Event-driven |
| 복잡한 조합 | Sequential + Conditional 조합 |
