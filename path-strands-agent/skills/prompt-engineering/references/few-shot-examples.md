# Few-shot Prompting (예시 기반 프롬프팅)

## 개념

Few-shot prompting은 프롬프트에 몇 가지 예시를 포함하여 모델이 원하는 형식과 스타일로 응답하도록 유도하는 기법입니다.

## 기본 구조

```
[지시문]

예시 1:
입력: [예시 입력 1]
출력: [예시 출력 1]

예시 2:
입력: [예시 입력 2]
출력: [예시 출력 2]

실제 입력: [실제 입력]
출력:
```

## 예시 개수 가이드

| 작업 복잡도 | 권장 예시 개수 |
|------------|---------------|
| 단순 분류 | 2-3개 |
| 형식 변환 | 2-3개 |
| 복잡한 분석 | 3-5개 |
| 창작/생성 | 1-2개 |

## 분류 작업 예시

### 감정 분석

```
다음 리뷰의 감정을 분류하세요.

예시 1:
리뷰: "배송이 빠르고 제품 품질이 좋아요!"
감정: positive

예시 2:
리뷰: "가격은 괜찮은데 배송이 너무 늦었어요."
감정: mixed

예시 3:
리뷰: "제품이 설명과 완전 달라요. 환불 요청합니다."
감정: negative

실제 리뷰: "디자인은 예쁜데 크기가 생각보다 작네요"
감정:
```

### 카테고리 분류

```
고객 문의를 적절한 부서로 라우팅하세요.

예시 1:
문의: "주문한 상품이 아직 안 왔어요"
부서: shipping

예시 2:
문의: "카드 결제가 두 번 됐어요"
부서: billing

예시 3:
문의: "앱이 자꾸 꺼져요"
부서: support

실제 문의: "제품 사용법을 모르겠어요"
부서:
```

## 형식 변환 예시

### JSON 추출

```
텍스트에서 정보를 추출하여 JSON으로 변환하세요.

예시 1:
텍스트: "김철수씨는 35세이며 서울에 거주합니다."
JSON: {"name": "김철수", "age": 35, "city": "서울"}

예시 2:
텍스트: "이영희(28세)는 부산 출신입니다."
JSON: {"name": "이영희", "age": 28, "city": "부산"}

실제 텍스트: "박지민, 42세, 대전 거주"
JSON:
```

### 요약 형식

```
기사를 3줄로 요약하세요.

예시:
기사: "애플이 신제품 아이폰 15를 발표했다. 새로운 칩셋과 개선된 카메라가 특징이다. 가격은 전작과 비슷한 수준이다. 9월 22일 출시 예정이며 사전예약은 다음 주부터 시작된다."
요약:
1. 애플, 아이폰 15 신제품 발표
2. 새 칩셋과 개선된 카메라 탑재
3. 9월 22일 출시, 다음 주 사전예약 시작

실제 기사: [기사 내용]
요약:
```

## 분석 작업 예시

### 장단점 분석

```
제품 리뷰에서 장단점을 추출하세요.

예시:
리뷰: "화질이 정말 좋고 색감도 자연스러워요. 다만 무게가 좀 무겁고 배터리가 빨리 닳는 게 아쉽네요."
분석:
- 장점: 화질 우수, 자연스러운 색감
- 단점: 무거움, 배터리 소모 빠름

실제 리뷰: [리뷰 내용]
분석:
```

### 의도 파악

```
사용자 발화의 의도를 파악하세요.

예시 1:
발화: "내일 서울 날씨 어때?"
의도: weather_inquiry
엔티티: {"location": "서울", "date": "tomorrow"}

예시 2:
발화: "피자 두 판 주문해줘"
의도: order_food
엔티티: {"item": "피자", "quantity": 2}

실제 발화: "다음 주 부산 가는 기차표 있어?"
의도:
엔티티:
```

## 생성 작업 예시

### 스타일 변환

```
문장을 더 격식 있게 바꾸세요.

예시 1:
원문: "빨리 답장 줘"
변환: "신속한 회신 부탁드립니다."

예시 2:
원문: "이거 어떻게 써?"
변환: "사용 방법을 안내해 주시겠습니까?"

실제 원문: "그거 언제 되는데?"
변환:
```

### 제목 생성

```
기사 본문을 보고 클릭을 유도하는 제목을 작성하세요.

예시 1:
본문: "삼성전자가 새로운 폴더블 폰을 출시했다..."
제목: "삼성, '접는 혁신' 신제품 공개...경쟁사 긴장"

예시 2:
본문: "정부가 내년 최저임금을 5% 인상한다고..."
제목: "최저임금 5% 오른다...자영업계 '한숨'"

실제 본문: [본문]
제목:
```

## 예시 선택 팁

### 다양성 확보

```
좋은 예시 세트:
- 긍정 케이스
- 부정 케이스
- 경계 케이스 (애매한 상황)
- 예외 케이스
```

### 난이도 순서

```
쉬운 예시 → 어려운 예시 → 실제 작업

모델이 패턴을 점진적으로 학습하도록 배열
```

### 대표성

```
실제 데이터와 유사한 예시 선택
- 길이 비슷
- 어휘 수준 비슷
- 구조 비슷
```

## 주의사항

1. **예시 편향 방지**: 한 카테고리에 치우치지 않도록
2. **최신 정보**: 예시가 구식 정보를 포함하지 않도록
3. **명확한 패턴**: 모델이 학습할 패턴이 명확하도록
4. **토큰 효율**: 너무 많은 예시는 컨텍스트 낭비
5. **질 > 양**: 적은 수의 좋은 예시가 많은 나쁜 예시보다 효과적
